# SkillGate / 被動收入專案優先級規則 v0.1

## 原則
這不是一般待辦清單，而是 CEO 式的資源分配規則。

第一優先順序固定為：
1. **增加收入**
2. **提高自動化程度**
3. **降低時間浪費**
4. **降低風險與成本**
5. **建立可複製系統**
6. **建立自己的護城河**

每個任務先看四件事：
1. **ROI**：能不能直接帶來收入、節省成本、或提高轉換？
2. **可複製性**：能不能重複用、委派、產品化、模板化？
3. **可自動化**：能不能變成 SOP、腳本、系統、工具？
4. **護城河價值**：能不能強化 SkillGate、內容資產、數據資產、案例資產、市場位置？

如果這四件事都不漂亮，優先級就別裝高。

## P1 — 直接影響收入或產品護城河
適用於：
- SkillGate 核心規格
- Threat Intel Sources
- `PERMISSIONS.md` 標準
- Diff Gate
- 入庫報告包
- 可直接變產品、服務、lead magnet 的內容
- 能直接影響成交、定價、留存、差異化的工作

處理原則：
- 優先做
- 持續追
- 需要時拆子任務並行
- heartbeat 優先檢查與續跑

## P2 — 放大效率的系統建設
適用於：
- SOP
- 自動化流程
- 研究與蒐集管線
- 模板系統
- 任務分派與回報機制
- 可把 1 次工作變 10 次可複用的建設

處理原則：
- 緊跟 P1
- 能省時間、減人力、加速產出者優先

## P3 — 支援型工作
適用於：
- 一般研究
- 文件整理
- 次要優化
- 補充性分析

處理原則：
- 不阻塞 P1 / P2 時再做
- 可批次處理，不要切碎主線注意力

## P4 — 低 ROI 雜務
適用於：
- 一次性小修小補
- 不能累積資產的零散工作
- 沒有明顯收入、效率、護城河效果的事

處理原則：
- 能砍就砍
- 能延就延
- 能自動化就不要手做
- 能不做就別做

## 決策規則
### 核心決策原則
- 沒有明確收益路徑的事情，先不優先。
- 不能複製擴張的事情，降級處理。
- 高風險低回報的事情，直接拒絕。
- 先做 MVP，再擴大。
- 先求穩定現金流，再追高想像力專案。

### 直接升 P1 的條件
- 直接影響現金流
- 直接影響 SkillGate 核心能力
- 直接影響產品差異化與市場敘事
- 直接產出可賣內容、模板、產品功能

### 直接降級的條件
- 沒有明確產出
- 沒有可重複價值
- 不能放大
- 只是看起來忙

## Heartbeat 執行規則
heartbeat 遇到多個未完成任務時，優先順序是：
1. P1 + 高時效
2. P1 + 高槓桿
3. P2 + 可自動化
4. P3 批次處理
5. P4 盡量不碰

heartbeat 不只是提醒未完成，而是要把最值得做的未完成任務往前推一步。

## 回報格式
預設回報格式固定為：
1. 現況摘要
2. 問題
3. 建議
4. 立即下一步
5. 預期商業價值

## 一句話版本
先做會賺錢、會放大、會累積資產的事；其他的，不要演得好像很重要。
